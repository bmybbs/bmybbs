# 模块 ID

> 状态：初稿

本页面预留了供用例和实现计划等结构化 Wiki 文档使用的模块/领域 ID。

## 已分配的 ID

| ID | 模块 / 领域 | 备注 |
| --- | --- | --- |
| `00` | 日志系统 | 遗留 `newtrace`、语义化日志 API、PostgreSQL 日志导入以及日志直接写入。 |

## 规则

- ID 是从 `00` 到 `FF` 的两位十六进制值。
- 在同一个模块/领域的用例和实现计划中使用相同的 ID。
- 在为新模块/领域创建结构化文档之前，请先添加新行。
- 不要将 ID 重用于不同的模块/领域。
- 如果某个模块/领域需要超过 10,000 份同类型的文档，请分配另一个 ID 并在备注中说明划分情况。
