# 日志系统第三阶段：直接数据库写入

> 状态：初稿

## 摘要

第三阶段将生产环境的写入路径从仅限文件的日志记录转向直接数据库写入。由于本阶段取决于第二阶段的模式和导入器经验，目前有意保持简略。

## 预期方向

- 使用第一阶段的语义化日志接口作为写入边界。
- 使用第二阶段的类别模式作为数据库支持的写入基础。
- 在部署新路径的同时，保持导入工具可用于历史补全、回放或切换支持。
- 稍后决定 `bbslogd` 是仅保留用于运行时诊断、进一步收缩还是退役。

## 开放性问题

- 哪些类别应优先支持直接数据库写入？
- 在过渡窗口期间，直接写入是否也应保持文件日志记录？
- 在哪个时间点 `bbslogd` 应停止作为活跃的业务日志接收器？
