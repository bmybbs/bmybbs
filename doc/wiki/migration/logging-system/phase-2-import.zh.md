# 日志系统第二阶段：数据库设计与历史导入

> 状态：初稿

## 摘要

第二阶段将在第一阶段理清日志类别后设计数据库结构。它还将为现有的基于文件的 `newtrace` 日志引入历史导入功能。

本页面目前是在日志系统迁移页面拆分期间创建的阶段占位符。在拆分评审通过后，应在单独的后续草稿中添加详细的模式 (schema) 设计。

## 预览

- 在第一阶段理清日志类别后设计数据库结构。
- 为现有的基于文件的日志引入日志导入器。
- 导入器应在每次调用时接受一个日志文件路径，并仅处理该文件。
- 批量导入应在以后通过重复调用导入器的脚本完成，而非通过一次性的单体导入任务。
- 在直接数据库写入部署到生产环境之前，导入器在过渡期内应保持有用。

## 备注

- 保持本页面专注于第二阶段。
- 将类别和模式设计的变更作为单独的修订进行评审。
