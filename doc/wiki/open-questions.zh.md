# 开放性问题

> 状态：初稿

## 摘要

本页面追踪从当前规划角度来看仍真正处于开放状态的重要问题。它并不旨在重复其他页面中已经提到的每一处局部不确定性。相反，它侧重于那些可能在架构、迁移成本、实现策略或项目规划方面产生实质性影响的问题。

## 架构问题

### 迁移后的存储成本

- 问题：
  - 在切换到 PostgreSQL 和对象存储之后，新系统总共需要多少存储空间？
- 为何重要：
  - 存储成本会影响基础设施规划、备份策略，以及项目对于在保留 GBK 数据的同时存储 UTF-8 数据的容忍度。
  - 新设计可能会通过压缩或附件重用在某些领域减少空间，但双格式存储也可能增加总容量。
- 相关页面：
  - `project-overview.md`
  - `target-architecture.md`
- 当前状态：
  - 开放 (Open)。

## 子系统问题

### TUI 实现成本

- 问题：
  - 设计和实现一个实用的 TUI 客户端需要多少时间和精力？
- 为何重要：
  - 未来的 TUI 是目标方向的一部分，但当前团队在 `ratatui` 和相关 Rust UI 工具方面的经验有限。
  - 这种不确定性可能会影响进度安排、子系统边界，以及项目在早期迁移阶段对 TUI 的依赖程度。
- 相关页面：
  - `project-overview.md`
  - `target-architecture.md`
- 当前状态：
  - 开放 (Open)。
